

# 오디오 임베딩 및 메타데이터 생성 파이프라인

## 개요

본 코드는 원시 오디오(.wav) 데이터를 임베딩 벡터로 변환하고, 이를 기반으로 머신러닝 학습에 사용할 JSON 메타데이터를 생성하는 전처리 파이프라인이다.

전체 과정은 다음 3단계로 구성된다:

1. 오디오 → 임베딩 추출
2. 임베딩 → `.npy` 파일 저장
3. 파일 구조 → JSON 메타데이터 생성

---

## 1. 오디오 임베딩 추출

`.wav` 파일을 입력으로 받아 CLAP 기반 모델을 통해 고정 차원 벡터로 변환한다.

### 주요 처리 과정

* torchaudio를 이용한 오디오 로딩
* stereo → mono 변환
* sampling rate 정규화
* 길이 보정 (padding)
* mel spectrogram 변환
* PyTorch 기반 CLAP 모델로 embedding 추출

### 저장 구조

```text
embeddings/
  ├── playlist_id/
  │     ├── track_id.npy
  │     └── ...
```

각 `.npy` 파일은 하나의 트랙에 대한 임베딩 벡터를 의미한다.

---

## 2. 단일 임베딩 메타데이터 생성

임베딩 파일들을 기반으로 간단한 JSON 메타데이터를 생성한다.

### 예시

```json
{
  "track_id": "12345",
  "audio_path": ".../12345.npy"
}
```

### 목적

* 전체 트랙 catalog 구성
* 모델 입력용 embedding 경로 관리

---

## 3. 플레이리스트 기반 메타데이터 생성

플레이리스트 구조를 반영한 JSON 메타데이터를 생성한다.

### 예시

```json
{
  "playlist_id": "100",
  "track_id": "12345",
  "audio_path": ".../12345.npy",
  "gen_path": ".../12345_gen.npy"
}
```

### 특징

* 플레이리스트 단위 데이터 구성
* 원본 임베딩(`audio_path`)과 생성 임베딩(`gen_path`)을 함께 관리
* 생성 임베딩이 없는 경우 `null` 처리

---

## 4. 전체 데이터 흐름

본 파이프라인은 다음과 같은 흐름으로 구성된다:

```text
.wav → 임베딩 추출 → .npy 저장 → JSON 메타데이터 생성
```

이를 통해 머신러닝 모델 학습에 필요한 입력 데이터를 효율적으로 구성할 수 있다.

---

## 5. 활용 목적

* 오디오 데이터를 고정 차원 벡터로 변환
* 대규모 데이터에 대한 빠른 입출력 지원
* 추천 모델 학습을 위한 데이터 구조 구성
